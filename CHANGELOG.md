# Changelog

Все заметные изменения проекта фиксируются в этом файле.

Формат основан на Keep a Changelog, версии ведутся по SemVer.

## [Unreleased]

### Fixed

- Исправлено обновление portable-лаунчера: новый exe скачивается рядом с запущенным лаунчером, выбирается portable asset и после закрытия приложения заменяет старый файл.
- Отключено лишнее выделение текста и текстовый курсор в интерфейсе, при этом поля ввода сохраняют выделение и курсор ввода.

### Changed

- Стандартная ширина окна лаунчера увеличена до 1580 пикселей.

## [1.0.5] - 2026-06-15

### Changed

- Изменена стандартная высота окна лаунчера до 820 пикселей.

## [1.0.4] - 2026-06-15

### Added

- Добавлена вкладка `Майнинг` с явным согласием пользователя, выбором exe-файла майнера, видимыми аргументами запуска, запуском/остановкой процесса, остановкой перед запуском игры и дашбордом hashrate, shares и локальной статистики.

### Fixed

- Улучшена читаемость текста и placeholder в полях ввода на стеклянном фоне интерфейса.
- Улучшен контраст зеленых и предупреждающих статус-бейджей в таблицах и панелях.
- Улучшен контраст всех основных кнопок, dropdown-кнопок, пунктов меню и ссылок-кнопок на стеклянных панелях.
- Во вкладку `Майнинг` добавлены примеры кошельков ETH/BTC/LTC/ETC для подстановки в форму, свернутый гайд по скачиванию майнера и ссылки на официальные источники популярных майнеров.
- Добавлены плавные переходы между вкладками рабочей области лаунчера.

## [1.0.3] - 2026-06-15

### Changed

- Выпущена техническая пересборка лаунчера для проверки нового release build.

## [1.0.2] - 2026-06-15

### Added

- Добавлено самообновление лаунчера: проверка GitHub Releases, выбор Windows installer asset, скачивание обновления и запуск установщика с закрытием приложения.
- Добавлено удаление аддонов из вкладки `Аддоны`: кнопка справа открывает модалку подтверждения и удаляет папки аддона из `Interface/AddOns`.
- Проверка обновлений лаунчера использует сохраненный GitHub token только как fallback для приватного репозитория, публичные релизы проверяются без token.

### Fixed

- Исправлено отображение вкладки аддонов после стартовой проверки и обновления: экран теперь хранит полный каталог отдельно от результата проверки и больше не скрывает неустановленные аддоны.
- Исправлен счетчик `К обновлению` во вкладке аддонов: неустановленные аддоны больше не считаются обновлениями.
- Для аддонов со статусом `Git, вручную` отключена кнопка удаления, чтобы лаунчер не трогал папки, управляемые пользователем.
- Проверка обновлений лаунчера больше не показывает ошибку при 404 от GitHub Releases API, а считает обновления временно недоступными.
- Исправлена установка аддонов, у которых `.toc` лежит в корне GitHub-репозитория: такие zip теперь ставятся в папку из каталога, например `AeonoPlates`.

### Changed

- Каталог community-аддонов пересобран из `Mr-Dan/AddonUpdaterSettings`, удалено неиспользуемое поле `sources` и добавлен недостающий `AeonoPlates`.
- Встроенный каталог аддонов отсортирован по имени внутри source-групп.

## [1.0.1] - 2026-06-15

### Added

- Добавлена вкладка `Сказать спасибо` с карточками поддержки fxpw и accidev через GitHub, Boosty и ЮMoney.
- Добавлена проверка актуальности FPS-патча по удаленным метаданным: размеру файла и дате обновления на зеркале.

### Changed

- GitHub Actions теперь создает GitHub Release по версии из `VERSION`, прикладывает Windows artifacts с предсказуемыми именами и использует заметки из `CHANGELOG.md` для обновлений через релизы.

## [1.0.0] - 2026-06-15

### Added

- Добавлен стартовый план разработки лаунчера.
- Добавлены правила разработки для backend/frontend и project skills в `.agents`.
- Добавлен будущий модуль автообновления WeakAuras в план.
- Зафиксировано правило установки GitHub-аддонов через zip архива кода репозитория.
- Добавлен файл `VERSION` для релизного версионирования.
- Добавлен GitHub Actions workflow для будущего autobuild.
- Добавлен changelog skill для обязательного обновления `CHANGELOG.md` агентами.
- Добавлен commit skill для описания коммитов агентами.
- Добавлен план модуля мультиаккаунтов с локальными профилями и подстановкой аккаунта в `WTF/Config.wtf`.
- План проекта возвращен на Electron + Vue 3 + TypeScript backend.
- Добавлено требование к тестируемому portable core, чтобы будущий переход на другой desktop shell был быстрым.
- Добавлен план механики автообновления лаунчера через проверку версии в GitHub Releases и скачивание релизного артефакта.
- Добавлен план настройки GitHub token для скачивания аддонов с повышенными лимитами.
- Добавлен стартовый Electron/Vue/TypeScript каркас приложения.
- Добавлены portable `core`-модули для `Config.wtf`, GitHub token/rate-limit, source zip URL и проверки версии лаунчера.
- Добавлены unit tests и portability test для запрета Electron imports в `core`.
- Добавлено зашифрованное хранение GitHub token через Electron `safeStorage`.
- GitHub Actions теперь собирает Electron output и на push может упаковать release artifacts через electron-builder.
- Добавлен документ чекпоинтов с текущим состоянием реализации и списком следующих задач.
- Расширены npm scripts и README-команды для dev, typecheck, tests, build и packaging.
- Добавлены persistent settings, выбор папки WoW через Electron dialog и UI для сохранения/удаления GitHub token.
- Frontend разнесен по архитектуре `page -> block -> element -> component`.
- Добавлены локализация `ru/en` и темная тема интерфейса.
- IPC handlers переведены на schema-first Zod validation для входов и выходов.
- Добавлен модуль создания backup `WTF`: core-план, zip adapter, IPC list/create, UI-блок и интеграционный тест.
- Добавлено восстановление, удаление и открытие папки backup `WTF` с safety-бекапом перед restore и защитой zip-распаковки от выхода за целевую папку.
- Добавлена установка FPS-патча: скачивание во временный файл, fallback на альтернативную ссылку, установка в `Data/ruRU`, IPC status/install, UI-кнопка install/reinstall и тесты.
- Добавлена проверка клиента по manifest Sirus: fallback API-источники, безопасное построение локальных путей, stream MD5, IPC `client.check`, UI-сводка и тесты.
- Добавлены настройки форматирования проекта через `.editorconfig` и Prettier с табуляцией шириной 4.
- Обновлена целевая среда сборки до Node.js 22.22.3 и npm 11.17.0; Windows packaging проверен на `electron-builder` 26.15.3.
- Переработан frontend: главная стала сводкой с плитками состояния, левое меню разделено на аддоны, проверку клиента, патч, WTF и настройки, интерфейс переведен на glass-style палитру лаунчера.
- Добавлена модалка первого запуска на главной для быстрого выбора папки клиента WoW, если путь еще не задан.
- Перенастроена цветовая схема glass-интерфейса: темная палитра стала основной, холодное стекло отделено от оранжевых акцентов.
- Исправлен выбор папки клиента в dev-режиме: системный dialog теперь привязан к окну приложения, а модалка пути показывается при пустом пути в любом разделе.
- Добавлена защита renderer API от отсутствующего Electron preload, чтобы вместо падения `window.launcher.settings` показывалась понятная ошибка.
- Добавлены автор `fxpw accidev` и иконка приложения для Windows-сборки.
- Обновлена палитра интерфейса по изображению с бордовым знаком, графитовым фоном, светлой обводкой и двумя гармоничными акцентами.
- В проверку клиента добавлена загрузка manifest-файлов, таблица проверяемых файлов и скачивание одного или всех проблемных файлов с MD5-валидацией.
- Статус GitHub token в шапке стал кликабельным и открывает быструю модалку сохранения token.
- Добавлен persistent cache MD5 файлов клиента по пути, размеру и времени изменения для ускорения повторных проверок.
- Интерфейс разделен на закрепленные header, левое меню, footer/statusbar и прокручиваемое рабочее окно.
- На главную страницу добавлены быстрые действия: выбор папки, запуск игры, проверка клиента, переход к аддонам, WTF-бекап, установка и удаление FPS-патча.
- Добавлен запуск игры через проверенный `run.exe` выбранного клиента WoW.
- Кнопки быстрых действий получили lucide-иконки, footer получил icon-only запуск игры, а основная цветовая роль кнопок заменена с красной на спокойную teal-палитру.
- На странице FPS-патча статус перенесен в строку действий, добавлены иконки и отдельная красная кнопка удаления установленного патча.
- В проверку клиента добавлен выбор источника manifest-файлов из доступных зеркал.
- Добавлена базовая поддержка мультиаккаунтов: dropdown в header, модалка добавления логина/пароля, локальное хранение и подстановка выбранного аккаунта в `WTF/Config.wtf` перед запуском.

### Changed

- Интерфейс переведен на единственную темную тему без переключателя светлой темы.
- Проверка папки клиента WoW теперь ищет `run.exe` вместо `Wow.exe`.
- Пользовательские подписи проверки клиента больше не называют раздел `MD5 клиента`.
- Каталог аддонов на странице `Аддоны` разделен на три явные таблицы: Sirus, сообщество и пользовательские ссылки.
- Плитки главной страницы сделаны шире по умолчанию и раньше переходят в двухколоночную сетку.
- Дата последнего WTF-бекапа на главной теперь показывается с годом.
- Скролл рабочего окна сделан более минималистичным и полупрозрачным.
- Сводка проверки клиента перенесена вверх: показываются `Проверено: x/y`, количество проблемных файлов и общее число файлов для проверки.
- Размеры файлов в проверке клиента теперь показываются в MB или GB.
- Выбор источника проверки клиента заменен на кастомный glass-dropdown без системного синего стиля.
- Dropdown аккаунта в header перенесен вправо и больше не обрезается верхней панелью.
- На главной странице плитки `Аддоны` и `FPS-патч` поменяны местами.
- Выбор языка в header заменен на кастомный glass-dropdown.
- Кнопки языка и аккаунта в header выровнены по высоте с кнопкой GitHub token.
- Стандартный размер окна приложения изменен на `1500x720`.
- Нижняя строка таблицы проверки клиента с `Проблемных файлов` и `Скачать все` удалена.
- При запуске приложения автоматически стартует проверка клиента, если сохраненный путь к WoW валиден.
- Фон приложения теперь генерируется при запуске renderer-а в отдельный canvas-слой из SVG с двумя левыми облаками Акацуки и запоминается на время работы приложения.
- В модалку GitHub token добавлен гайд по созданию fine-grained token для публичных репозиториев аддонов без лишних прав, только для повышения лимитов GitHub.
- Добавлен модуль аддонов: встроенный JSON-каталог community/Sirus, локальные custom GitHub-ссылки, проверка `.toc` версий, установка source zip и обновление всех подходящих аддонов.
- Для пользовательских аддонов добавлены локальное per-user хранение, экспорт и импорт custom GitHub-ссылок через JSON.
- Сравнение версий аддонов теперь учитывает SemVer, dotted numeric, plain numeric и строковый fallback для `.toc` строк вида `## Version: 5.21.6`.
- Кнопка проверки аддонов теперь проверяет только установленные аддоны, найденные по совпадению имени папки с каталогом.
- В таблице аддонов добавлена hover-подсказка с причиной текущего статуса.
- При запуске лаунчер сканирует установленные аддоны, показывает счетчик доступных обновлений и умеет автообновлять их по новой настройке.

### Fixed

- Исправлена ошибка компиляции Vue в dropdown аккаунта из-за многострочного inline-обработчика.
- Исправлен путь к Electron preload bundle от корня приложения, из-за которого в dev-режиме renderer не видел `window.launcher`.
- Исправлена установка FPS-патча, если временная папка лаунчера и папка WoW находятся на разных дисках Windows.
- Исправлена обработка действий WTF-бекапов: кнопка создания теперь показывает состояние выполнения, блокирует повторный запуск и выводит ошибку в интерфейсе.
- Исправлена прокрутка правого рабочего окна и уменьшена высота закрепленного header.
- Выровнен правый край рабочего окна с header и footer.
- Исправлена прокрутка разделов настроек и проверки клиента внутри закрепленного рабочего окна: панели больше не ужимаются и не обрезают содержимое.
- Аддоны, установленные как git-репозитории, теперь определяются по папке `.git` и пропускаются автообновлением.
- Исправлено исчезновение неустановленных аддонов из таблиц после проверки или массового обновления аддонов.
- Исправлен выбор локальной версии для пакетных аддонов вроде WeakAuras: версия теперь берется из основной папки аддона, а не из первой вспомогательной папки.
- Исправлен каталог WeakAuras: удалена несуществующая папка `WeakAurasSounds`, из-за которой аддон считался устаревшим при одинаковых версиях.
- Исправлен каталог EPGP: базовый `EPGP` и отдельный `EPGP_Auction` больше не притягивают папки друг друга.
- Исправлена упаковка в GitHub Actions: `electron-builder` больше не пытается автоматически публиковать release без `GH_TOKEN`.
- Для пакетных аддонов добавлены явные `versionFolder`/`versionFile`; исправлены проверки версий и списки папок DBM и Details.
